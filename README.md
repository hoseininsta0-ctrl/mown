# موان (Mown)

### واسط دانلود با زیرساخت گیت‌هاب

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Next.js](https://img.shields.io/badge/Next.js-16-black)](https://nextjs.org)

---

> **هدف پروژه:** موان برای شرایطی ساخته شده که دسترسی به گیت‌هاب وجود دارد اما سایت‌های دیگر (مثل یوتیوب) مسدود هستند. با استفاده از GitHub Actions به‌عنوان زیرساخت، فایل‌ها را بدون نیاز به VPN دانلود می‌کند.

---

## ✨ قابلیت‌ها

- **دانلود یوتیوب** — انتخاب کیفیت (360p تا 4K)، ویدیو (MP4/WebM) یا صدا (MP3)
- **دانلود مستقیم URL** — دریافت هر فایل عمومی از اینترنت
- **اسنپ‌شات صفحه وب** — ذخیره کامل یک صفحه به‌صورت آفلاین در قالب HTML تک‌فایل (با [monolith](https://github.com/Y2Z/monolith))
- **تقسیم خودکار فایل‌های بزرگ** — فایل‌های بیشتر از ۹۵ مگابایت به‌صورت خودکار به چانک‌های ZIP تقسیم می‌شوند
- **تاریخچه دانلودها** — ردیابی همه job ها با وضعیت، لاگ و لینک دانلود مجدد
- **ذخیره رمزنگاری‌شده کوکی** — کوکی‌های یوتیوب با libsodium رمزنگاری و به‌عنوان Actions Secret ذخیره می‌شوند
- **ویزارد راه‌اندازی** — راهنمای گام‌به‌گام برای کاربران جدید
- **پشتیبانی از RTL و چندزبانه** — رابط کاربری کامل به فارسی و انگلیسی
- **تم تاریک / روشن**
- **بدون نیاز به سرور** — کاملاً روی زیرساخت رایگان گیت‌هاب اجرا می‌شود

---

## 🏗️ نحوه عملکرد

```
کاربر URL را ارسال می‌کند
          │
          ▼
    موان (اپ Next.js)
  ├─ توکن و تنظیمات را در localStorage ذخیره می‌کند (هرگز به سرور ارسال نمی‌شود)
  ├─ GitHub API را صدا می‌زند تا یک workflow راه‌اندازی کند
          │
          ▼
  GitHub Actions (در مخزن شخصی شما)
  ├─ youtube-download.yml  →  yt-dlp + ffmpeg
  ├─ direct-download.yml   →  curl
  └─ snapshot.yml          →  monolith (Rust)
          │
          ▼
  آرتیفکت آپلود می‌شود (نگهداری ۱ روز) + commit به مخزن شما
          │
          ▼
موان وضعیت اجرا را پیگیری می‌کند → لاگ نمایش می‌دهد → لینک دانلود ارائه می‌دهد
```

توکن گیت‌هاب و کوکی‌های یوتیوب **هرگز از مرورگر شما خارج نمی‌شوند** — مستقیماً به GitHub API ارسال می‌شوند.

## 📝 پیش‌نیازها

۱. یک **حساب گیت‌هاب** (پلن رایگان کافی است)  
۲. یک **مرورگر مبتنی بر Chromium** (Chrome، Edge، Brave و …) — فقط برای استخراج کوکی‌های یوتیوب

---

## 🚀 راهنمای راه‌اندازی

### مرحله ۱ — ساخت توکن شخصی گیت‌هاب

۱. به [github.com](https://github.com) بروید و وارد حساب کاربری خود شوید  
۲. روی عکس پروفایل کلیک کنید ← **Settings**  
۳. از منوی چپ: **Developer settings** ← **Personal access tokens** ← **Tokens (classic)**  
۴. روی **Generate new token (classic)** کلیک کنید  
۵. یک نام مثل `mown` وارد کنید  
۶. ⚠️ حتماً scope های **`repo`** و **`workflow`** را فعال کنید  
۷. روی **Generate token** کلیک کنید و توکن را کپی کنید — دیگر قابل مشاهده نخواهد بود

> توکن را مثل رمز عبور اصلی حسابتان نگه دارید.

---

### مرحله ۲ — دریافت کوکی‌های یوتیوب (اختیاری، برای ویدیوهای با محدودیت سنی)

۱. افزونه [Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/cclelndahbckbenkjhflpdbgdldlbecc) را در مرورگر نصب کنید  
۲. به [youtube.com](https://youtube.com) بروید و وارد حساب یوتیوب خود شوید  
۳. روی آیکون افزونه کلیک کنید ← **Export**  
۴. محتوای فایل `cookies.txt` را در تنظیمات موان پیست کنید

---

### مرحله ۳ — راه‌اندازی مخزن

۱. اپ موان را باز کنید  
۲. توکن گیت‌هاب را در پنل تنظیمات وارد کنید  
۳. روی **راه‌اندازی مخزن** کلیک کنید — موان به‌صورت خودکار:

- یک مخزن خصوصی `mown-downloads` در حساب شما می‌سازد
- سه فایل workflow را در `.github/workflows/` آپلود می‌کند
- کوکی‌های یوتیوب را به‌عنوان Actions Secret رمزنگاری‌شده ذخیره می‌کند

---

## 📖 نحوه استفاده

### دانلود ویدیو

۱. لینک ویدیوی یوتیوب را در کادر ورودی پیست کنید  
۲. کیفیت (`بهترین`، `1080p`، `720p`، `480p` و …) و فرمت (`MP4` / `MP3`) را انتخاب کنید  
۳. روی **شروع دانلود** کلیک کنید  
۴. پیشرفت را در لحظه دنبال کنید — لاگ‌ها از GitHub Actions stream می‌شوند  
۵. پس از اتمام، فایل در بخش **تاریخچه** قابل دانلود است

### فایل‌های بزرگ (بیشتر از ۹۵ مگابایت)

فایل‌های بیشتر از ۹۵ مگابایت به‌صورت خودکار به چانک‌های ZIP شماره‌گذاری‌شده تقسیم می‌شوند.  
همه پارت‌ها را دانلود کنید، سپس اولین `.zip` را extract کنید — ویدیو به‌صورت خودکار از پارت‌ها بازسازی می‌شود.

### مدیریت دانلودها

در بخش **تاریخچه** می‌توانید:

- هر آرتیفکت را مجدد دانلود کنید (تا ۱ روز پس از دانلود)
- فایل‌های commit‌شده را مستقیماً در مخزن گیت‌هاب مشاهده کنید
- commit دقیق هر دانلود را ببینید

## 🛠️ اجرا در محیط توسعه

```bash
# کلون کردن مخزن
git clone https://github.com/mr-meshky/mown.git
cd mown

# نصب وابستگی‌ها
pnpm install

# اجرای سرور توسعه
pnpm dev
```

آدرس [http://localhost:3000](http://localhost:3000) را در مرورگر باز کنید.

### بیلد برای محیط پروداکشن

```bash
pnpm build
pnpm start
```

## 🛡️ امنیت

| چه چیزی            | کجا ذخیره می‌شود                         | چه کسی می‌بیند     |
| ------------------ | ---------------------------------------- | ------------------ |
| توکن گیت‌هاب       | `localStorage` مرورگر شما                | فقط شما            |
| کوکی‌های یوتیوب    | Actions Secret رمزنگاری‌شده با libsodium | فقط runner گیت‌هاب |
| فایل‌های دانلودشده | مخزن خصوصی گیت‌هاب + آرتیفکت موقت        | فقط شما            |

**بهترین شیوه‌ها:**

- هرگز توکن یا فایل کوکی را به اشتراک نگذارید
- اگر نشت توکن مشکوک شدید، فوراً آن را در GitHub Settings ← Developer settings باطل کنید
- فقط روی دستگاه شخصی خودتان استفاده کنید

## 🔧 عیب‌یابی

| مشکل                         | راه‌حل                                                                                 |
| ---------------------------- | -------------------------------------------------------------------------------------- |
| دانلود شروع نمی‌شود          | بررسی کنید توکن scope های `repo` و `workflow` را داشته باشد و مخزن راه‌اندازی شده باشد |

| فایل بزرگ بازسازی نمی‌شود    | مطمئن شوید **همه** پارت‌های شماره‌دار را قبل از extract دانلود کرده‌اید                |
| کندی workflow اسنپ‌شات       | `cargo install monolith` از سورس کامپایل می‌شود — اجرای اول حدود ۳ دقیقه طول می‌کشد    |
